# AI 分层记忆设计

## 1. 目标

JPQuiz 中的 AI 不再是一次性问答框，而是围绕“当前课次 + 当前模块 + 学习者进度 + 最近对话”提供稳定辅导。

首期目标：

- 支持多轮追问，不再只有 1 轮记忆
- 控制上下文体积，避免无上限堆历史消息
- 让 AI 明确知道自己是日语学习助手，而不是泛用聊天机器人
- 为后续“长期记忆”“AI 主动整理薄弱本”“AI 自主沉淀题库”预留数据层与配置层
- 让每位学习者未来都能拥有独立的 Tutor 配置、会话和记忆

非目标：

- 首期不做无限长对话记忆
- 首期不把所有 prompt 完全开放给用户编辑
- 首期不让 AI 直接执行高风险自主操作

## 2. Prompt 分层

Tutor 问答统一采用以下 4 层上下文。

### 2.1 Base Safety Prompt

内置、不可编辑、必须始终生效。

职责：

- 明确 AI 是《大家的日语》N5 学习助手
- 限制回答范围聚焦日语学习
- 拒绝黄赌毒、政治、宗教、战争、暴力、违法、自残、极端主义等敏感内容
- 阻止借“词汇学习”“翻译”“例句”名义绕开安全限制

### 2.2 Tutor Profile Prompt

账号级可配置。

职责：

- 助手名字
- 教学风格
- 输出偏好
- 个性化导师要求

示例：

- 助手名字：`みな先生`
- 风格：`structured`
- 输出偏好：`table-first`
- 个性化要求：动词讲解优先输出原型、类别、礼貌体、中文

### 2.3 Learner Snapshot Prompt

由系统自动生成，不手写。

职责：

- 当前课次与模块
- 当前课掌握度
- 当前薄弱项
- 最近错题
- 最近测验摘要
- 今日学习时长

这层只传压缩后的结构化学习状态，不传完整原始日志。

### 2.4 Task Context Prompt

按问题动态注入。

职责：

- 当前课次、当前模块的内容摘要
- 追问时引用当前任务的最近结果
- 只带和本次回答真正相关的教材内容

示例：

- 问“总结第 7 课动词”时，优先带入第 7 课词汇摘要
- 问“再加上中文意思”时，优先延续上一轮任务，而不是重新生成无关结果

## 3. 记忆分层

## 3.1 工作记忆

作用：服务“当前任务连续追问”。

特点：

- 生命周期短
- 强相关
- 解决“加一列中文意思就完全换答案”这类问题

首期状态：

- 用“最近几轮对话 + 当前任务上下文”替代显式 artifact memory
- 后续再升级为结构化任务态

## 3.2 会话记忆

作用：保留最近多轮问答。

特点：

- 只保留最近 `N` 轮
- 历史消息超过阈值后压缩为摘要
- 不把整段长历史原文全部回放给模型

首期状态：

- 已落表 `aiConversations`、`aiMessages`、`aiConversationSummaries`
- 当前实现为“最近几轮 + 本地压缩摘要”

## 3.3 长期记忆

作用：保留稳定偏好与稳定事实。

适合沉淀：

- 输出偏好
- 关注重点
- 高频薄弱点
- 学习目标

不适合沉淀：

- 一次性闲聊
- 临时上下文
- 无法稳定验证的推测

首期状态：

- 已预留 `aiLongTermMemories` 表
- 后续再接入“何时晋升长期记忆”的策略

## 4. Token 预算原则

默认策略不是“尽量多带”，而是“尽量只带必要内容”。

推荐优先级：

1. 安全基座
2. 当前任务最相关的课内内容
3. 最近几轮对话
4. 学习者快照
5. 少量长期记忆

默认预算建议：

- 安全基座：固定短 prompt
- 最近对话：`recentTurns`
- 薄弱项：`weakItemsLimit`
- 最近错题：`recentWrongAnswersLimit`
- 当前模块内容条数：`moduleContextItemsLimit`
- 长期记忆条数：`maxLongTermMemoriesPerRequest`

所有这些都应通过配置可调，而不是写死在多个文件里。

## 5. 哪些可配置，哪些不可配置

### 可配置

- 助手名字
- 教学风格
- 输出格式偏好
- 自定义导师提示词
- 最近保留轮数
- 薄弱项数量
- 最近错题数量
- 总结触发轮数
- 长期记忆条数
- 上下文 Token 软上限
- 模块内容摘要条数

### 不可配置

- 基础安全提示词
- 结构化 Quiz JSON 规则
- 目标识别器与组卷器的核心格式约束

原因：

- 安全边界不能依赖用户配置
- 结构化功能需要稳定格式，不能被随意改坏

## 6. 数据层设计

当前 AI 相关新增表：

- `aiConversations`
- `aiMessages`
- `aiConversationSummaries`
- `aiLongTermMemories`

设计原则：

- 未来每个账号独立隔离
- 当前单机版使用默认 `ownerId=local-default`
- 后续上云时可平滑迁移到真正用户体系

## 7. 首期实现范围

本阶段先完成：

- Tutor Prompt 分层
- 导师配置与记忆策略配置
- 本地多轮会话
- 最近几轮记忆注入
- 当前课次学习快照注入
- 当前模块内容摘要注入
- 基础摘要能力

暂不在本阶段完成：

- 长期记忆自动晋升
- AI 主动改写薄弱本
- AI 自主存储题库
- Tool Use 调度

## 8. 后续演进方向

### 8.1 结构化工作记忆

为高频任务维护 artifact，例如：

- 当前课动词表
- 当前语法对比表
- 当前错题讲解列表

这样后续“加列”“改排序”“补中文意思”可以直接在 artifact 上增量修改。

### 8.2 长期记忆晋升策略

基于重复出现次数、时间跨度、学习行为稳定性，把偏好和薄弱点写入长期记忆。

### 8.3 Tool Use

后续可逐步开放安全范围内的工具能力，例如：

- AI 根据错题自动整理薄弱本
- AI 将高质量练习题沉淀到题库
- AI 对学习日志做定期汇总

前提：

- 必须明确写权限边界
- 必须保留可审计记录
- 必须允许用户关闭或回滚
