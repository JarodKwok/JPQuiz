# 大家的日语 AI陪练 PRD

## 一、文档概述

### 1.1 产品名称

大家的日语 AI陪练

### 1.2 产品定位

一个围绕《大家的日语》初级 I（第 1 课～第 25 课）构建的 Web 学习工具，提供：

- AI 生成学习内容
- AI 智能组卷测验
- 学习进度与掌握度跟踪
- 错题 / 薄弱项强化
- AI 学情总结与复习建议

### 1.3 目标用户

- 正在学习《大家的日语》的 N5 初学者
- 需要按课次、按专题系统复习的学习者
- 希望在学习后快速自测，并持续追踪掌握进度的用户

### 1.4 核心价值

- 不需要手动整理教材资料，AI 自动生成当前课次学习内容
- 每个专题模块同时支持“学习”与“测验”双模式
- 测验题型固定、UI 原生化，告别聊天式作答
- 支持按当前范围、指定目标、错题 / 薄弱项、混合强化进行智能组卷
- 系统持续记录学习表现，并基于统计结果生成 AI 学情点评

### 1.5 运行平台

- Web 端（桌面浏览器优先）
- 移动端响应式适配

## 三、全局功能需求

### 3.1 顶部状态栏

- 展示产品名称
- 展示当前课次
- 展示各专题掌握度百分比
- 展示今日学习时长
- 提供“复习薄弱”快捷入口

### 3.2 课次选择

- 支持第 1 课～第 25 课切换
- 切换课次后，当前专题页自动切换到对应课次内容
- 测验模式和学习模式都遵循当前课次上下文

### 3.3 左侧专题导航

- 单词
- 语法
- 课文
- 例句
- 听力
- 薄弱本
- 学习记录

### 3.4 底部 AI 问答栏

底部 AI 输入栏保留，但职责调整为：

- 自由问答
- 知识讲解
- 学习答疑
- 解释错题
- 补充说明

**不作为正式结构化测验的主入口。**

正式测验统一在专题页的“测验”tab 内完成。

### 3.5 掌握状态标记

所有专题学习内容都保留三种掌握状态：

- ✅ 会了
- ⚠️ 模糊
- ❌ 不会

这些状态会参与：

- 模块掌握度统计
- 薄弱项汇总
- 智能组卷中的“错题 / 薄弱项”题源

## 四、专题模块需求

每个专题模块都必须包含两个子模式：

- `学习`
- `测验`

### 4.1 单词模块

#### 学习模式

- 展示单词
- 展示读音
- 展示中文释义
- 展示例句
- 支持发音（Edge TTS 拟人化神经网络男声，内容命名体系，与顺序解耦）
- 支持掌握度标记（会了 / 模糊 / 不会）
- 支持按掌握状态筛选：全部 / 不会 / 模糊 / 待复习（不会 + 模糊 + 未标记），各分类显示数量
- 工具栏（序号 / 汉字 / 中文开关）滚动时悬浮置顶，方便全局控制

#### 测验模式

支持题型：

- 选择题
- 填空题
- 问答题（翻译）

支持组卷来源：

- 当前第 X 课单词随机出题
- 用户指定单词出题
- 按错题 / 薄弱词汇出题
- 混合强化组卷

用户可用中文、日文、假名指定目标，例如：

- 老师
- 先生
- せんせい

系统会先展示识别到的目标词条，再正式生成题目。

### 4.2 语法模块

#### 学习模式

- AI 列出当前课次语法点
- 展示含义、接续、例句、易错点
- 支持掌握度标记

#### 测验模式

支持题型：

- 选择题：语法辨析、助词选择
- 填空题：句型补全、接续补全
- 问答题（翻译）：按语法翻译句子

支持组卷来源：

- 当前语法范围随机
- 指定语法点
- 薄弱语法点
- 混合强化

### 4.3 课文模块

#### 学习模式

- 展示课文标题
- 展示课文原文
- 展示逐句翻译
- 展示备注信息
- 支持整篇朗读
- 支持掌握度标记

#### 测验模式

支持题型：

- 选择题：课文理解
- 填空题：句子挖空
- 问答题（翻译）：句子翻译

支持组卷来源：

- 当前课文随机
- 指定句子或片段
- 薄弱项
- 混合强化

### 4.4 例句模块

#### 学习模式

- 展示例句
- 展示读音
- 展示中文翻译
- 展示关联语法（如有）
- 支持发音
- 支持掌握度标记

#### 测验模式

支持题型：

- 选择题：句意 / 场景判断
- 填空题：例句补全
- 问答题（翻译）：例句翻译

支持组卷来源：

- 当前例句随机
- 指定例句
- 薄弱项
- 混合强化

### 4.5 听力模块

#### 学习模式

- 展示当前课次听力题
- 支持 TTS/播放
- 支持听选答案
- 自动记录作答表现

#### 测验模式

首版仍采用“文本 + 朗读辅助”的听力测验方式。

支持题型：

- 选择题
- 填空题
- 问答题（翻译）

支持组卷来源：

- 当前听力范围随机
- 指定目标
- 薄弱项
- 混合强化

### 4.6 听力训练功能（跨模块增强）

听力训练不作为独立模块，而是在现有专题模块（单词、语法、例句、课文）的学习模式中内嵌支持，通过控制文字可见性配合音频播放实现。

#### 设计理念

听力训练的核心流程：**盲听 → 看日文 → 看中文释义**

用户先隐藏所有文字，只通过音频播放来听，尝试理解内容；然后逐步显示日文原文验证听力理解；最后显示中文释义确认语义。

#### 功能要求

**全局控制按钮（工具栏）**

每个支持听力训练的模块在学习模式工具栏中提供：

- **日文开关**：一键隐藏/显示所有条目的日文文字（假名、汉字、句子原文等）
- **中文开关**：一键隐藏/显示所有条目的中文翻译/释义

两个开关独立控制，可组合使用：
- 日文 OFF + 中文 OFF → 纯听力模式（盲听）
- 日文 ON + 中文 OFF → 听读对照模式
- 日文 OFF + 中文 ON → 听译对照模式
- 日文 ON + 中文 ON → 完整学习模式（默认）

**单条控制按钮**

每个学习条目右侧提供独立的日文/中文切换按钮，允许用户针对单条内容单独控制显示状态，不影响其他条目。

**各模块适配说明**

| 模块 | 日文开关控制范围 | 中文开关控制范围 |
|------|----------------|----------------|
| 单词 | 假名（word） | 中文释义（meaning）、例句 |
| 语法 | — | — |
| 例句 | 日文原句（japanese）、读音（reading）、内嵌句型例句 | 中文翻译（translation）、句型释义（meaning）、句型例句翻译 |
| 课文 | — | 逐句翻译 |

> 语法和课文模块的日文控制可在后续迭代中补充。

**交互细节**

- 隐藏时显示占位符文字（如 `[日文已隐藏]`、`[中文已隐藏]`、`[释义已隐藏]`），保持布局稳定
- 全局开关切换时，所有条目状态同步更新
- 单条切换不影响全局开关状态和其他条目
- 音频播放按钮始终可见，不受文字隐藏影响

## 五、智能组卷系统

### 5.1 出题来源

测验模式必须支持以下四种题源：

1. `当前范围随机`
2. `指定目标`
3. `错题 / 薄弱项`
4. `混合强化`

### 5.2 指定目标输入

在“指定目标”或“混合强化”模式下，用户可通过自然语言输入目标内容。

支持输入：

- 中文词义
- 日文词汇
- 假名
- 语法关键词
- 课文句子片段
- 例句片段

### 5.3 目标识别流程

目标输入流程必须为：

1. 用户输入目标文本
2. 系统基于当前课次 + 当前专题识别候选知识点
3. 页面展示已识别目标
4. 用户确认后再生成题目

### 5.4 混合强化策略

混合强化模式下，系统优先顺序为：

1. 用户手动指定目标
2. 当前错题 / 薄弱项
3. 当前课次专题范围随机内容

## 六、测验题型标准

### 6.1 选择题

- 单题 4 个选项
- 单选
- 可直接本地判题
- 展示标准答案与解析

### 6.2 填空题

- 首版统一为单空题
- AI 提供标准答案与可接受答案
- 前端本地做文本规范化判题
- 展示标准答案与解析

### 6.3 问答题（翻译）

- 用户输入完整答案
- AI 提供标准答案与可接受答案
- 前端本地做规则匹配判题
- 首版不做开放式语义评分

## 七、页面交互要求

### 7.1 专题页统一结构

每个专题页统一结构：

1. 页面标题与课次信息
2. `学习 / 测验` 双 tab
3. 学习模式内容区
4. 测验模式组卷区
5. 测验模式答题区

### 7.2 测验模式统一 UI

测验 tab 内需要统一包含：

- 出题来源选择
- 目标输入区（视来源显示）
- 已识别目标列表
- 题型选择
- 题量选择
- 生成测验按钮
- 测验题目卡片
- 提交答案按钮
- 逐题解析与结果总结

### 7.3 标准使用场景

#### 场景 A：第 5 课单词定向出题

1. 用户先学习第 5 课单词
2. 进入 `单词 > 测验`
3. 选择 `指定目标`
4. 输入：`老师、先生、銀行`
5. 系统识别目标词条
6. 用户选择 `5 道选择题`
7. 系统生成 5 道结构化题目
8. 用户直接在页面中点击选项提交
9. 系统展示正确率与解析
10. 错题与掌握状态同步更新

#### 场景 B：按薄弱项出题

1. 用户先在学习或测验中积累若干错题 / 弱项
2. 进入任一专题的 `测验`
3. 选择 `错题 / 薄弱项`
4. 选择题型和题量
5. 系统基于薄弱内容自动组卷
6. 用户完成作答
7. 系统更新掌握度、错题与测验记录

### 8.1 记录维度

系统需记录：

- 已学课次
- 各专题掌握度
- 各知识点掌握状态
- 总学习时长
- 错题数量
- 测验场次

学习记录页支持生成 AI 学情点评，输入应基于系统统计结果而不是纯聊天历史。

## 十一、音频系统设计

### 11.1 音频架构概览

| 模块 | 命名体系 | 存储位置 | 服务方式 | 声音策略 |
|------|---------|----------|---------|---------|
| 单词（vocab） | 内容命名：`vocab_{读音}.mp3` | `books/audio/lesson_XX/` | `/api/audio/books/` API 路由 | 男声（KeitaNeural） |
| 课文（text） | 索引命名：`text_000.mp3` | `public/audio/lessons/lesson_XX/` | Next.js 静态文件 | 按说话人性别：女→NanamiNeural，男→KeitaNeural |
| 例句（example） | 索引命名：`example_000.mp3` | `public/audio/lessons/lesson_XX/` | Next.js 静态文件 | 男声（KeitaNeural） |
| 听力（listening） | 索引命名：`listening_000.mp3` | `public/audio/lessons/lesson_XX/` | Next.js 静态文件 | 男声（KeitaNeural） |

### 11.2 音频声音规格

- **男声（词汇、例句、听力默认）**：`ja-JP-KeitaNeural`（Microsoft Edge TTS 高质量神经网络男声）
- **女声（课文女性角色）**：`ja-JP-NanamiNeural`（Microsoft Edge TTS 高质量神经网络女声）
- **语速**：`-10%`（略慢，适合学习朗读）
- **格式**：MP3

### 11.3 课文说话人声音分配规则

课文对话中，根据说话人姓名自动分配男女声：

- **已知女性角色**：カリナ、ワン、佐藤、店員(女)、マリア、テレーザ、アンナ → 女声
- **已知男性角色**：ミラー、グプタ、山田、サントス、木村、先生 等 → 男声
- **名字特征**：姓名末尾含子/美/恵/江/代/香/奈/理/花/葉 → 女声
- **其他/未知**：默认男声

### 11.4 单词音频命名设计（内容命名体系）

单词音频采用**基于读音的内容命名**，与词汇在课次中的排列顺序完全解耦：

```
books/audio/lesson_01/vocab_わたし.mp3
books/audio/lesson_01/vocab_がくせい.mp3
books/audio/lesson_01/vocab_〜じん.mp3   → 文件名: vocab_じん.mp3 (去掉~)
```

**优势**：
- 教材内容重新排序后，音频文件无需重建，自动匹配
- 新增词汇只需生成新文件，不影响已有文件
- 便于逐词独立替换或更新音质

### 11.5 音频生成脚本

| 脚本 | 用途 | 输出目录 |
|------|------|---------|
| `scripts/migrate_vocab_audio.py` | **优先复用**旧音频，对新增词汇调用 TTS 生成 | `books/audio/` |
| `scripts/generate_books_audio.py` | 全量重新生成词汇音频（内容命名体系） | `books/audio/` |
| `scripts/generate_edge_tts.py` | 生成课文/例句/听力音频（索引命名体系），支持 `--overwrite` | `public/audio/lessons/` |

**推荐工作流**：

```bash
# 激活环境
source venv313/bin/activate

# 步骤1：迁移词汇音频（复用旧文件 + 生成新文件）
python scripts/migrate_vocab_audio.py

# 步骤2：重新生成课文/例句音频（内容更新后）
python scripts/generate_edge_tts.py --overwrite

# 单独重建某课词汇
python scripts/migrate_vocab_audio.py --lesson 3 --overwrite
```

### 11.6 前端音频服务

```typescript
// 单词 → 内容命名，走 API 路由
speakVocab(text, lessonId, reading)
// URL 示例：/api/audio/books/lesson_01/vocab_わたし.mp3

// 课文/例句/听力 → 索引命名，走 Next.js 静态文件
speak(text, lessonId, type, itemIndex)
// URL 示例：/audio/lessons/lesson_01/text_000.mp3
```

三级降级策略：
1. 预生成 MP3（Edge TTS 拟人化神经网络声音）
2. 浏览器 Web Speech API（系统 TTS，音质较差）
3. 静默（无法播放时不报错）

### 11.7 音频文件 Git 策略

音频文件体积大，不纳入 Git 版本管理：

```gitignore
/books/audio/        # 词汇音频（内容命名体系）
/public/audio/       # 课文/例句音频（索引命名体系）
```

部署时通过脚本在本地或 CI 生成。
